Introduction
1.1
Machine Learning Problems
Machine learning is, and has been for the past couple of decades, one of the most exciting
research areas of computer science and engineering. The idea that a computer software could
have the ability to learn and improve performance without being explicitly programmed to
do, does not only come with a academic research interest but also vast applications within a
wide range of industries. Machine learning problems are usually classified into the following
3 different categories:
1. Unsupervised learning
2. Supervised learning
(a) Regression
(b) Classification
3. Reinforcement learning
These problems arise due to different availability of feedback.
Unsupervised learning methods assume no feedback from the system. This means that
no labels are given to learning algorithm. The system must detect the important features
and hidden patters of the data, solely based on the input.
Supervised learning techniques are used when the data are given not only the inputs but
also the desired outputs. In other words, given ’labelled’ observations x1, x2, ..., xn and their
corresponding response variables y1, ..., yn, we observe {(xi, yi)}n
i=1.Thegoalofsupervised
learning is to correctly predict outcomes yi for new inputs xi. There are typically two types
of supervised learning problems. Classification problems arise when we deal with discrete
outcomes, e.g. Y ∈ {−1, 1} or
{1, ..., K}. Regression problems arise when the outcome
is numerical and continuous, Y ∈ R.
Reinforcement learning occurs when computer program interacts with a dynamic envi-
ronment in which it must perform a certain goal. Reinforcement learning is concerned with
how an agent can perform actions to receive feedback from the environment. The feedback
3
is provided in terms of reward or punishment from the dynamic environment. Reinforce-
ment learning differs from supervised learning. In supervised learning we build machine
learning models based on a training set where all the outcomes are known. It is often very
difficult with a dynamic environment to be able to decide whether a certain behaviour of
the agent is correct, hence reinforcement learning, an agent must be able to learn from its
own experience.
1.2
Background
Ever since the first computers were created, the ultimate aim of artificial intelligence has
been to build machines which can perform tasks better than humans. From Charles Bab-
bage’s mechanical computer to IBM’s Deep Blue being the first computer system to beat
a reigning chess world champion, computer scientists have proved that machines can be
specifically tailored to perform certain tasks more accurately and efficiently than humans.
However, a truly intelligent software would be able to learn from its mistakes and improve
performance towards a certain goal, without actually being programmed to do so. Deep
Blue was a computer software built specifically to play chess, Turing’s original Enigma de-
crypting machine was built specifically to aid mathematicians in code decryption. Could a
universal algorithm be used in different objective scenarios in order to receive feedback and
improve its own performance?
Over the past decade, companies in various fields such as fintech and healthcare have in-
vested into reinforcement learning research as a potential self-learning data analytic tool.
One of the most exciting companies currently working on deep reinforcement algorithms is
Deepmind. Similarly to many artificial intelligence research groups, Deepmind believe that
these algorithms can be implemented in order to understand and control the behaviour of
complex multi-agent systems such as the economy, traffic, and environmental challenges. So
far the algorithms which are used from Deepmind or other projects such as Google Brain
have been efficient when used towards reaching different objectives. The main issue with this
ideology has been that building one algorithm is never going to emulate the human brain.
The human brain is complicated and different parts of the brain work on specific tasks. An
interesting experiment from [Roe et al., 1992] gave birth to the so called ’One Algorithm’
hypothesis. The ”input” for vision within a group of ferrets was plugged into auditory part
of brain in order to observe whether the auditory cortex of the ferrets could learn to ”see”.
These experiments were successful and gave evidence to suggest that the human brain uses
essentially the same algorithm to understand many different input modalities.
One of the challenges that arise in reinforcement learning and not in other kinds of learning
is the trade-off between exploration and exploitation [Sutton and Barto, 1998]. In order to
obtain a higher reward and reach its objective, an agent must prefer actions and decisions
which it has made in the past. Nevertheless, exploration should be an essential feature
of these agents. Exploration could help these agents discover more efficient methods of
performing certain tasks.

n-Armed Bandit Problem
Consider the following: an agent at a certain point in time is faced with n possible actions to
take. Each action comes with a numerical reward from a stationary probability distribution.
The agent must take a certain pre-specified number of these decisions and its objective is to
maximise the final reward. The analogy of an ’n-armed bandit’ comes from slot machines -
one armed bandits, except the machine has n levers. Multi-armed bandit problems are the
most basic examples of sequential decision problems with an exploration-exploitation trade-
off. This is the balance between staying with the option that gave highest pay-offs in the
past and exploring new options that might give higher pay-offs in the future [Bubeck et al.,
2012]. Each time the agent performs the task the rewards from each action are rewarded
and each action will have an expected reward associated with it, called the value of that
action. Since the rewards from each action come from a stationary probability distribution,
there is always some uncertainty in the reward estimates of the agent. If the agent continues
to maintain and upgrade action values, there will be at any time at least one action whose
estimated value is greatest which we call the greedy action. Choosing a greedy action means
the agent is exploiting the current knowledge of the action values. Choosing any other
action means the agent is exploring. Exploring is important because it improves the value
estimates of the non-greedy actions.

Environment Interaction
Every machine learning model consists of a learner which we call the agent and the thing
it interacts with which compromises everything outside the agent called the environment.
The agent and the environment interact at every discrete time step2 t = 1, 2, 3,
At each

At each time point t, there exists a mapping from states to probabilities. This assigns

The value function and the action-value function can be estimated by Monte Carlo non-

In section 1.2 we defined a greedy action a∗. An agent which only chooses greedy actions
does not explore the remaining actions and might miss out on a higher final return. A simple
method for ensuring adequate exploration is by assigning some level of stochasticity to the
agent’s decisions. The agent can behave greedily most of the time but with a probability
ϵ it will explore the other actions at random. Methods which use this are called ϵ-greedy
methods. These methods rise a number of concerns such as which value of ϵ to use and that
the greedy methods actually perform significantly better at first in terms of accumulated
reward. However these near-greedy methods outperform the greedy methods in the long
run as the greedy agents tend to get stuck performing suboptimal actions.

Neural networks are discriminative learning methods used in machine learning. Neural
networks are widely used in supervised learning problems for classification or regression.

Chapter 4
Conclusions And Further Work
We have presented in this paper an introduction to Deep Reinforcement Learning and some
important concepts and identities which are essential in reinforcement learning algorithms.
Let us summarise the reinforcement learning elements we have presented in this topic.
Chapter one of this project gives an introduction to the importance of Bayesian methods
and Markov Decision Processes in reinforcement learning problems where environment in-
teraction is essential. The reinforcement learning agent and its environment interact over a
sequence of discrete time steps. The agent interacts with the environment by performing a
pre-defined task where it chooses actions at different states of the environment. After each
action taken the agent is given a reward which it then uses to evaluate the choice of actions
in the future. The agent is given an objective function which it tries to maximise.
An important concept introduced in this chapter is the exploration-exploitation trade-off
problem which is typically mitigated by ϵ-greedy policies. The ϵ-greedy methods choose
random actions a small fraction of the time, and the softmax methods grade their action
probabilities according to the current action-value estimates. An interesting policy which
we addressed in this paper was to use a non-constant value of ϵ throughout training. Since
the agent seems to learn how to play the game fairly quickly, we decided that exploration
should not have the same probability for latter training epochs. We adapted the training
code in order to decrease the value of ϵ as the training progresses. This led to a higher
accumulated reward for training as shown in figure 3.5.
These ϵ-greedy algorithms were used throughout this project, however there are other poli-
cies we could consider further. One promising idea mentioned by [Sutton and Barto, 1998]
is to use estimates of the uncertainty of the action-value estimates to direct and encourage
exploration. Consider two non-greedy actions to have estimated values slightly less than
the greedy action. These actions however differ in uncertainty. The estimated value for
one action is nearly certain due to the action being previously chosen many times, whilst
the other action is well known and has a high uncertainty in its value prediction. The true
value of the second action could be greater than the greedy action, hence it makes more
sense to explore this action compared to the one with a low uncertainty.
In section 3.1.3 we discussed the difficulties which arise when building deep neural networks.
We decided that building a network as complex as the one used by Deepmind was not
necessary for our simple game. Hence we used a two-layer network with a convolutional
21
input layer. The difficulties in this case arise when choosing the layer sizes for the hidden
layers where some trial and error exploration is necessary. In figure 3.6 we can see that
the model with 100 neurons in the hidden layer performs better than others. However we
usually need to address the question whether an increase in model complexity is worth the
increase in computational time. In this project we have used the same number of neurons
in each hidden layer which could not be the best performing network possible.
Multi-agent environments are interesting extensions to the Deep Q-Learning model we have
used in this chapter. Different agents are controlled by autonomous Deep Q-Networks and
interact when sharing a complex environment. Multi-agent systems appear naturally in
social, economic or political scenarios. An interesting paper which observes the effects of two
agents playing a game of Pong whilst using independent Deep Q-networks is [Tampuu et al.,
2015]. This paper uses game theory principles to assign rewards and objective functions to
each agent. I am currently using these principles to build a multi-agent based model of the
economy which takes into account different population needs based on a geographical scale
